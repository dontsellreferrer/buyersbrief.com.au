## Step 3: Create Cloudflare DNS Record

Once you have the Railway domain, create a CNAME record in Cloudflare:

### How to add CNAME in Cloudflare:

1. Go to https://dash.cloudflare.com
2. Select your domain: **buyersbrief.com.au**
3. Go to **DNS** tab
4. Click **"Add record"**
5. Fill in:
   - **Type:** CNAME
   - **Name:** @ (or buyersbrief.com.au)
   - **Target:** (Railway domain, e.g., `buyersbrief-production.up.railway.app`)
   - **TTL:** Auto
   - **Proxy status:** Proxied (orange cloud)
6. Click **"Save"**

**DNS will propagate in 5-30 minutes.** Once propagated, buyersbrief.com.au will point to your Railway deployment.

## Deployment Workflow

### Automatic Deployments

Every time you push code to GitHub `main` branch, Railway automatically:

1. Detects the push
2. Pulls the latest code
3. Installs dependencies (`pnpm install`)
4. Builds the project (`pnpm build`)
5. Starts the server (`pnpm start`)
6. Deploys to production

**Time:** Usually 3-5 minutes per deployment

### Manual Deployments

To manually trigger a deployment:

1. Go to your Railway project
2. Click **"Deploy"** button (top-right)
3. Select the commit to deploy
4. Click **"Deploy"**

### Rollback to Previous Deployment

If a deployment breaks:

1. Go to **"Deployments"** tab
2. Find the previous working deployment
3. Click the **"..."** menu
4. Select **"Rollback"**

When you update the database schema:

2. **Generate migration:** `pnpm drizzle-kit generate`
3. **Apply migration:** Run the generated SQL via Supabase UI
4. **Commit & push:** Git push triggers Railway deployment

## Troubleshooting

### Build Fails

**Error:** `pnpm install` fails  
**Solution:** Check `package.json` for typos. Run locally first: `pnpm install`

**Error:** `pnpm build` fails  
**Solution:** Check TypeScript errors: `pnpm check`. Fix and commit.

### App Crashes

**Error:** `DATABASE_URL not set`  
**Solution:** Add DATABASE_URL to Railway Variables tab.

**Error:** `OPENAI_API_KEY invalid`  
**Solution:** Verify API key is correct. Check OpenAI dashboard.

### DNS Not Working

**Error:** buyersbrief.com.au shows "Connection refused"  
**Solution:** Wait for DNS propagation (5-30 min). Check Cloudflare DNS record is correct.

**Error:** SSL certificate error  
**Solution:** Cloudflare should auto-issue SSL. Wait 5 minutes and refresh.

## Backup & Recovery

### Database Backups

Supabase automatically backs up your database daily. To restore:

1. Go to Supabase dashboard
2. Go to **Settings → Backups**
3. Select a backup point
4. Click **"Restore"**

### Code Rollback

If code breaks production:

1. Go to Railway **Deployments** tab
2. Find the last working deployment
3. Click **"Rollback"**
4. Or manually revert on GitHub and push
